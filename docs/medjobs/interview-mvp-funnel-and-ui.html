MedJobs — Interview-Scheduling MVP

Funnel analysis + low-fidelity UI sketches for alignment. Scope: standardize on the existing (Gen 1) interview-scheduling system, open the loop (no paywall) for MVP, and hide the post-job / inbox (Gen 2) surfaces.

Terminal state = confirmed interview + calendar invite (.ics) on BOTH calendars Deferred: offer→placement→hire, CRM reconciliation

Part 1 — Funnel Analysis

Two funnels (provider acquisition is admin-CRM-fed; both sides self-serve from sign-in onward) converging on one shared object — the interviews table. The terminal-positive event is a confirmed interview, which fires dual calendar invites.
